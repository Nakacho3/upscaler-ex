# AIマスク生成機能 仕様書

## 1. 概要

本仕様書は、開発中のアップスケーリングアプリケーションに新規追加する「AIマスク生成（AI Mask Generation）」タブの機能要件・UI仕様・技術仕様を定義する。本機能は、ユーザーが読み込んだ画像に対し、GPU（NVIDIA RTXシリーズ）上で動作するAIセグメンテーションモデルを用いて画像解析を行い、人物・オブジェクト・背景などのカテゴリ別にマスクを自動生成するものである。

生成されたマスクは、アップスケーリング処理の対象範囲指定、領域別の画質設定（人物のみ高精細化、背景は軽量処理など）、または後続のレタッチ・合成作業のベースとして活用されることを想定する。

| 項目 | 内容 |
|---|---|
| 機能名 | AIマスク生成（仮称） |
| 対象 | アップスケーリングアプリ 新規タブ |
| 主要技術 | GPUベースAIセグメンテーション（RTXシリーズ対応） |
| バージョン | v0.1（初版ドラフト） |
| 作成日 | 2026-06-20 |

---

## 2. 背景・目的

アップスケーリング処理は画像全体に対して均一なアルゴリズムを適用すると、人物の肌や輪郭、テキスト、背景のテクスチャなどで最適なパラメータが異なり、仕上がりに差が出やすい。あらかじめ画像内の要素をカテゴリ別に分離したマスクを用意できれば、領域ごとに異なるアップスケール強度・ノイズ除去・シャープネスを適用でき、最終的な画質を向上させられる。

本機能の目的は以下の通りである。

1. ユーザーがワンクリックで画像内の主要要素（人物・オブジェクト・背景等）をAIに解析させ、領域別マスクを自動生成できるようにする。
2. RTXシリーズGPUのTensor Core / CUDAを活用し、ローカル環境で高速に推論を完了させる。
3. 生成したマスクをアップスケーリングのプリセット適用範囲として利用できるようにし、領域別の画質チューニングを可能にする。

## 3. 機能要件

### 3.1 タブ構成

既存のメインウィンドウに「マスク生成」タブ（仮称：**AI Mask**）を新設する。タブ内は大きく以下の3ブロックで構成する。

- 画像プレビュー＆マスクオーバーレイ表示エリア
- カテゴリ別マスクリスト（レイヤーパネル）
- 解析実行・パラメータ設定パネル

### 3.2 画像読み込み

- 既存のアップスケーリング機能で読み込んだ画像をそのまま本タブでも参照できること（画像の二重読み込みは不要）。
- 対応フォーマットは既存アプリの対応形式（PNG / JPEG / WebP / TIFF 等）に準拠する。
- 解析対象画像の解像度上限・下限は GPU VRAM 容量に応じて自動調整する（詳細は5章）。

### 3.3 AI解析（セグメンテーション）処理

- 「解析開始」ボタン押下時に、選択中の画像に対してGPU上でセグメンテーション推論を実行する。
- 推論には事前学習済みのセマンティックセグメンテーション／インスタンスセグメンテーションモデルを使用する（候補モデルは5.2節を参照）。
- 解析完了後、検出されたオブジェクトごとにマスク（アルファチャンネル付き2値またはグレースケール画像）を生成する。
- 同一カテゴリ内に複数インスタンスが存在する場合（人物が複数いる場合等）、個体ごとに分離したマスクを生成するか、カテゴリ統合マスクとして扱うかをユーザーが選択できる（3.5節参照）。

### 3.4 マスク分類カテゴリ

初期実装で対応するカテゴリは以下を基本セットとする。将来的にカスタムカテゴリ追加を可能とする拡張性を持たせる。

| カテゴリ | 説明 | 用途例 |
|---|---|---|
| 人物（Person） | 人物の輪郭全体を抽出 | 肌・顔の高精細アップスケール |
| 顔（Face）※サブカテゴリ | 人物マスクから顔領域のみを細分化 | 顔専用の高精細処理 |
| オブジェクト（Object） | 人物・背景以外の主要な物体（家具、動物、車両等） | 物体ごとのディテール強調 |
| テキスト・文字（Text） | ロゴ、文字、看板等 | 文字つぶれ防止処理 |
| 空・天候（Sky） | 空、雲等の領域 | ノイズ低減重視の処理 |
| 背景（Background） | 上記以外の背景領域 | 軽量・高速処理 |

- 各カテゴリは個別のON/OFF表示切替が可能。
- 検出信頼度（confidence score）をカテゴリ・インスタンスごとに保持し、UI上に表示する。

### 3.5 マスクの確認・編集

- 生成されたマスクはオーバーレイとして元画像上に半透明色分け表示する（カテゴリごとに色を割当）。
- マスク境界をブラシツールで微調整できる簡易編集機能を持つ（拡張・収縮、消去、追加描画）。
- 複数インスタンスの統合／分離をUI上で切り替え可能。
- マスクの信頼度が閾値以下の領域は警告表示し、ユーザーに手動確認を促す。

### 3.6 出力・連携

- 生成したマスクは以下の形式で出力・保存可能とする。
  - PNG（アルファチャンネル付き）
  - PSDレイヤー形式（オプション、対応ライブラリがある場合）
  - アプリ内部形式（アップスケーリング処理にそのまま引き渡すための内部データ）
- マスク単位でアップスケーリングのプリセット（強度・ノイズ除去・シャープネス等）を個別に割り当てられるようにし、アップスケーリング実行時にカテゴリ別パラメータを適用する。

### 4.2 主要UI要素

- **進捗表示**：推論の進行状況、推定残り時間、VRAM使用量をリアルタイム表示。
- **レイヤーパネル**：カテゴリ／インスタンス単位の表示切替、不透明度調整、削除・再生成ボタン。

## 5. 技術仕様

### 5.1 ハードウェア・GPU要件

| 項目 | 要件 |
|---|---|
| 対象GPU | NVIDIA RTXシリーズ（RTX 20/30/40/50系列を想定） |
| 必須VRAM | 最低6GB、推奨8GB以上（高解像度画像・複数カテゴリ同時推論時） |
| 推論基盤 | CUDA / cuDNN、可能であればTensorRTによる最適化 |
| 非対応時の挙動 | RTX非搭載環境ではCPUフォールバック（処理速度低下を明示）、またはCUDA対応GPUのみサポートと明記し非対応エラーを表示 |

### 5.2 AIモデル候補

セグメンテーションモデルは以下を候補とし、精度・速度・ライセンス条件を踏まえて選定する（実装フェーズで最終決定）。

- **汎用インスタンスセグメンテーション**：Segment Anything Model（SAM/SAM2）系
- **人物セグメンテーション特化**：人物・ポートレート向け軽量モデル（例：U^2-Net、MODNet等の背景分離特化モデル）
- **セマンティックセグメンテーション全般**：DeepLab系、Mask2Former系
- **顔検出サブカテゴリ**：顔検出・顔ランドモデルとの組み合わせ（人物マスク内からの二次抽出）

選定基準：推論速度（RTX上でのFPS／処理秒数）、VRAM消費量、商用利用ライセンス、カテゴリ拡張のしやすさ。

### 5.3 推論パイプライン（処理フロー）

1. 画像読み込み・前処理（リサイズ、正規化、GPUメモリへ転送）
2. モデル推論実行（CUDAストリーム上でセグメンテーション推論）
3. 後処理（マスクの2値化、輪郭平滑化、信頼度フィルタリング）
4. カテゴリ分類・インスタンス分離処理
5. マスクデータをUI表示用にCPU側へ転送・オーバーレイ生成
6. ユーザー編集（必要時）
7. マスクデータの保存・アップスケーリングパイプラインへの引き渡し

### 5.4 使用ライブラリ・SDK（想定）

- 推論ランタイム：ONNX Runtime（CUDA Execution Provider）または TensorRT
- 画像処理：OpenCV、NumPy相当の処理基盤
- GPU管理：CUDA Toolkit、NVMLによるVRAM/使用率モニタリング
- モデル形式：ONNX形式での配布・量子化（FP16/INT8）によるRTX最適化を検討

### 5.5 パフォーマンス要件

| 項目 | 目標値（参考） |
|---|---|
| 推論時間（1024×1024画像、RTX 4070相当） | 3秒以内 |
| 同時カテゴリ検出数 | 最大10カテゴリ／インスタンス |
| VRAM使用上限 | システム搭載VRAMの70%以内に制御 |
| UIフリーズ防止 | 推論は非同期処理（別スレッド／プロセス）で実行し、UIブロッキングを発生させない |

## 6. エラーハンドリング・例外処理

- CUDA非対応GPU検出時：機能を無効化し、対応GPU一覧へのリンクまたは説明をUI上に表示。
- VRAM不足時：画像の自動ダウンスケール推論＋結果のアップスケール戻し、またはエラーメッセージで解像度低下を提案。
- 推論失敗（モデル読み込み失敗等）：エラーログを出力し、再試行ボタンを表示。
- マスク検出ゼロ件（該当カテゴリなし）：「該当領域が検出されませんでした」を表示し、感度調整を促す。

---

## 7. 制約事項・前提条件

- 本機能はNVIDIA RTXシリーズ（CUDA対応）を前提とした最適化を行う。AMD/Intel GPUおよびApple Silicon環境への対応は本仕様の範囲外とし、別途検討する。
- 初版ではモデルはアプリにバンドルする事前学習済みモデルのみとし、ユーザーによるカスタムモデル読み込みは将来拡張とする。
- 動画ファイルへの対応は本仕様の対象外（静止画像のみ）。
