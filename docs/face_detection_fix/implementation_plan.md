# 顔検出閾値の調整による小さい人物の検出対応

人物像が小さい場合でも、人物の顔を確実に検出して顔復元処理を適用できるようにするため、`min_face_ratio` の閾値を調整します。

## 変更の目的と概要
`gfpgan_onnx.py` の `restore_faces` にて、画像面積に対する顔面積の割合が 3% 以上 (`min_face_ratio = 0.03`) である顔のみを復元対象とするフィルタリングが適用されています。この閾値を `0.001` (0.1%) に引き下げることで、引き気味の写真や人物が小さく写っている写真でも顔検出がスキップされずに修復されるようにします。

## 影響範囲
- 顔復元機能 (`gfpgan_onnx.py`) を使用する際の顔検出処理
- 人物が小さく写っている画像で、顔復元処理が適用されるようになります。

## 提案される変更

### 顔復元モジュール

#### [MODIFY] [gfpgan_onnx.py](file:///h:/My%20Apps/upscaler%20ex/gfpgan_onnx.py)

`gfpgan_onnx.py` の `min_face_ratio` の定義（181行目付近）を `0.03` から `0.001` に修正します。

```diff
-                     min_face_ratio = 0.03  # 画像面積の3%以上を占める顔のみ対象
+                     min_face_ratio = 0.001  # 画像面積の0.1%以上を占める顔のみ対象（小さい顔も検出可能にする）
```

## 検証計画

### 手動検証
1. アプリケーションを起動し、ユーザーから提供された人物像が小さい画像をロードします。
2. 「人物の顔・瞳を自然に修復」チェックボックスをONにして、高画質化処理を実行します。
3. ログおよびプレビュー画像を確認し、顔が検出され、修復処理が適用されることを確認します。
